# Skill Creator

This skill aids in the creation of new standardized skills for the Universal Skills system, ensuring consistency across all LLM interactions.

## When to Use
- When the user asks to create a new skill.
- When the user wants to document a process, technology key feature, or set of rules as a reusable skill.

## Instructions

1.  **Analyze User Request**: Identify if the user wants to create a new skill from scratch or convert existing documentation into a skill.
2.  **Reference Template**: Use the standard template located in `assets/skill-template.md` (relative to this skill's directory) as the strict structure.
3.  **Gather Necessary Information**:
    *   **Skill Name**: Should be kebab-case (e.g., `angular-best-practices`, `csharp-coding-standards`).
    *   **Description & Trigger**: Clearly define *when* this skill should be active.
    *   **Critical Patterns**: The non-negotiable rules or "Laws" of the skill.
    *   **Decision Tree**: A logic flow for common scenarios.
    *   **Examples**: Minimal but complete code examples.
4.  **Generate Output**: Create a new markdown file named `SKILL.md` inside a folder named `{skill-name}`.
5.  **Organization**: The structure should be `universal-skills/{skill-name}/SKILL.md`. If there are templates or assets, they should go in `universal-skills/{skill-name}/assets/`.

## Generation Rules:
Rule: Language Enforcement: Every output generated by this skill must be in English. If the user provides instructions or context in another language (e.g., Spanish), the agent must perform an internal translation before writing the final SKILL.md file.

## Goal
To produce high-quality, standardized context files that can be easily synchronized across Gemini, Claude, Cursor, and GitHub Copilot using the `sync-skills.ps1` script.
